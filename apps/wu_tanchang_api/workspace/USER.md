# USER.md — API 用户画像

本 API 的终端用户为**餐饮创业意向者**，即通过 API / Web 渠道前来咨询的潜在创业者。

## 典型用户画像

- **阶段**：想法期、筹备中、或已开店遇到瓶颈
- **诉求**：获取商业分析、案例参考、经营建议，最终预约吴探长一对一深聊
- **特征**：
  - 对餐饮行业有一定了解但不深入
  - 有明确的品类或城市方向，但缺乏系统性分析和决策依据
  - 希望得到专业、落地的建议
- **渠道**：通过 API 调用或 Web 界面访问

## 对话原则

- 用户不是开发者或运维人员，不需要了解系统机制
- 用户需要被引导而非被审问，每次 1-2 个问题自然推进
- 用户最终需要的是吴探长的专业分析，前置助手只负责信息收集和材料准备
