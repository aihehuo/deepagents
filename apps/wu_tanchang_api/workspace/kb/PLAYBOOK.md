# 吴探长 · 咨询回答 Playbook

## 触发条件

用户咨询涉及以下任一主题时，**必须先查知识库**（`kb/index.json` + chunks）：

- 餐饮开店 / 选址 / 选品 / 定价 / 客单
- 品牌定位 / 升级 / 连锁化 / 标准化
- 赛道趋势 / 竞品对标 / 单品店模型
- 具体品类：烘焙、茶饮、轻食、中式快餐、小吃等

## 回答流程

```
1. 识别问题类型（见下表）
2. grep kb/index.json —— 无命中则不得声称有该品牌探店笔记
3. 选 2–4 条【检索命中】的 entry，读取 chunks
4. 按 kb/METHOD.md 组织回答
5. 仅对命中品牌写「吴探长探店 XX 时…」
6. 库外品牌：明说「库内暂无」，改用库内相近案例或标注行业经验
7. 信息不足时追问
```

## 防幻觉检查（发送前自检）

- [ ] 提到的每个「探店案例」品牌，是否都在 `kb/index.json`？
- [ ] 具体数字是否来自刚读过的 chunk？否则删除或标「粗算」
- [ ] 用户要原文时，是否真的有 `kb/raw/` 文件？没有就说没有

## 问题类型 → 检索策略

| 类型 | 用户问法示例 | 优先 series | 检索字段 |
|------|-------------|-------------|----------|
| A 开店落地 | 「50万开烘焙店怎么做？」 | 商业探店笔记 | categories, consult_for |
| B 单品牌解析 | 「Punch Monday 为什么火？」 | 商业探店笔记 / 探店记 | brand, keywords |
| C 赛道趋势 | 「茶饮现在什么方向？」 | 市场调查 | topics, categories |
| D 品牌升级 | 「老字号怎么年轻化？」 | 案例说 | topics, consult_for |
| E 定价策略 | 「怎么做低价锚定？」 | 商业探店笔记 | topics 含「定价」「锚定」 |
| F 单品店模型 | 「贝果专门店能复制吗？」 | 探店记 | topics 含「单品店」 |

## 回答长度

| 场景 | 建议 |
|------|------|
| 飞书私聊 | 完整六段式，2–4 个案例，800–1500 字 |
| 飞书群聊 | 核心概述 + 1 个案例 + 追问，300–500 字 |

## 案例引用格式

> 吴探长在探店 **Punch Monday** 时看到：6.9 元做价格锚、12.9 元走量……

**前提：`Punch Monday` 必须在 `kb/index.json` 中能 grep 到。** 否则禁止使用该句式。

## 库外品牌 / 无笔记时

```
知识库目前没有【阿德生煎】的探店笔记。生煎赛道可参考库内：小杨生煎、苏小柳。
以下按行业经验粗算，非笔记原文——
```

不要编造未收录品牌的笔记内容或「刚才引用了笔记」。

## 检索命令（workspace 内）

```bash
# 按关键词搜 index
grep -i '烘焙' kb/index.json
grep -i '定价' kb/index.json

# 读某品牌 chunk
cat kb/chunks/brands/wu-punch-monday/dimension-*.md
```

或使用 skill：`wu-tanchang-kb`
